# ADR-009 — Tableau de bord et Chroniques vivantes

**Statut :** Acceptée

**Date :** 2026-07-07

## Contexte

Après la création fiable du Royaume via RPC, l'application doit donner une sensation de jeu dès l'arrivée sur la page d'accueil.

Un simple tableau de bord administratif ne correspond pas à la vision de HomeQuest.

## Décision

Le tableau de bord devient le premier écran vivant du Royaume.

Il affiche :

- le nom du Royaume ;
- la guilde familiale ;
- les statistiques simples ;
- les Domaines ;
- les Chroniques récentes ;
- la prochaine étape de jeu.

## Pourquoi

HomeQuest doit être perçu comme un RPG coopératif familial, pas comme une application de tâches.

La Chronique donne une dimension narrative : chaque action de la famille devient un souvenir du Royaume.

## MVP

Sprint 2.1 affiche :

- statistiques de base ;
- Domaine principal ;
- Chroniques récentes ;
- bouton de rafraîchissement ;
- déconnexion.

## Futur

Les prochains sprints ajouteront :

- quêtes actives ;
- boss actif ;
- progression des compétences ;
- carte visuelle des Domaines ;
- événements saisonniers.
